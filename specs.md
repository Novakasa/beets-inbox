# Music Importer — Specification

A self-hosted web app for importing music into a beets-managed library. All music flows through an **inbox** where the user reviews tags before committing. Packaged as a Nix flake with a NixOS module.

## Architecture

```
┌──────────────────────────┐   ┌────────────────────────┐
│  music-importer flake    │   │  external tools         │
│                          │   │  (ytdl-sub, scripts,   │
│  Web App (FastAPI + Elm) │   │   manual file drops)   │
│  Inbox beets library     │   └──────────┬─────────────┘
│  Main beets library      │              │
└────────┬─────────────────┘              │
         │                                │
  ┌──────▼──────┐              ┌──────────▼──────┐
  │  library/   │              │  inbox/          │
  │  (navidrome)│              │  {category}/     │
  └─────────────┘              │    files...      │
                               └─────────────────┘
```

**Web app** — Python (FastAPI) backend, Elm frontend. The only custom code.

**Two beets libraries:**
- **Inbox library** — scratchpad. Catalogs incoming files with auto-tagger (`copy: no`, `move: no`, `write: no`). Provides tag reading and MusicBrainz/bandcamp matching for free.
- **Main library** — the real library. Files are copied here with confirmed tags on import. Either generated by the module or an existing user-provided beets config.

**Inbox directory** — watched for new files. External tools (ytdl-sub, scripts) and the web UI's upload function all deposit files here.

**navidrome** — external, reads the library directory. Not managed by this app.

## Inbox Structure

Every file lives inside a **category** subdirectory. A configurable default category (e.g. `unsorted`) is used when none is specified. Within a category, standalone audio files are individual imports and subdirectories containing audio files are **album groups** imported as a unit.

```
{inboxPath}/
  unsorted/                                  # default category
    track.mp3
  dj-sets/                                   # category
    20240315 - DJ - Live Set.mp3
    20240315 - DJ - Live Set.mp3.info.json   # optional sidecar
  purchased/                                 # category
    Artist - Album/                          # album group
      01 - Track.flac
      02 - Track.flac
      cover.jpg
```

The category subdirectory name is pre-filled as a tag (e.g. genre or a custom beets field).

If a `.info.json` sidecar exists alongside an audio file (as produced by yt-dlp / ytdl-sub), the app parses it to pre-fill additional tag fields (uploader, title, source URL, upload date).

## Import Flow

### 1. File arrives in inbox
New files appear via web UI upload, ytdl-sub, scripts, or manual filesystem drops.

### 2. Automatic cataloging
A watcher detects new files and runs `beet --config inbox-beets.yaml import` — auto-tagger on, no copy/move/write. Beets reads embedded tags and attempts MusicBrainz matching. Results are stored in the inbox beets DB.

### 3. User review
The Inbox view queries the inbox beets DB to show pre-filled tags. The user reviews, corrects, and optionally re-triggers auto-matching.

### 4. Commit
On "Import":
1. `beet --config main-beets.yaml import -A --set artist="..." --set album="..." ... /path/to/file` — copies file to main library with confirmed tags
2. `beet --config inbox-beets.yaml remove -d path:/path/to/file` — removes inbox DB entry and deletes source file
3. Clean up sidecar files from inbox

Copy-then-remove ordering ensures no data loss on crash.

### Upload via Web UI
The Inbox view has an upload drop zone. User drags files or ZIPs, selects a category (or accepts the default). ZIPs are extracted — if they contain a top-level folder (as bandcamp ZIPs do), it becomes an album group. Files land in the inbox and enter the normal flow above.

## Tech Stack

### Backend: Python + FastAPI
- FastAPI for HTTP API
- `subprocess` calls to `beet` CLI for all beets operations
- SQLite (stdlib `sqlite3`) for job tracking
- `watchdog` or polling for inbox file detection
- Python 3.11+, mypy strict

### Frontend: Elm
- SPA served as static files by the backend
- Two views: **Inbox** (upload + tag review + import) and **Jobs** (import history/logs)
- Communicates with backend via JSON API

### API Endpoints
```
POST   /api/inbox/upload        Upload file(s) to inbox with optional category
GET    /api/inbox               List inbox items (all or filtered by category)
GET    /api/inbox/categories    List category subdirectories
POST   /api/inbox/{id}/import   Import with provided tags
DELETE /api/inbox/{id}          Discard an inbox item
GET    /api/jobs                List import jobs with status/logs
GET    /api/jobs/{id}           Get job details
```

### Job Model
```
Job:
  id: UUID
  status: "pending" | "running" | "success" | "failed"
  source_path: string
  category: string (optional)
  artist: string (optional)
  album: string (optional)
  genre: string (optional)
  log: string
  created_at: datetime
  completed_at: datetime (optional)
```

## NixOS Module

### Configuration

`libraryPath`/`beetsPlugins` and `beetsConfigPath` are mutually exclusive. The first generates a main library beets config; the second uses an existing one. The inbox library config is always generated by the app.

```nix
# Option A: app owns the main library
services.music-importer = {
  enable = true;
  inboxPath = "/mnt/nas/music-inbox";
  dataPath = "/var/lib/music-importer";
  port = 8085;
  defaultCategory = "unsorted";
  libraryPath = "/mnt/nas/music";
  beetsPlugins = [ "bandcamp" "fetchart" "embedart" ];
  user = "music-importer";
  group = "music-importer";
};

# Option B: use an existing beets config
services.music-importer = {
  enable = true;
  inboxPath = "/mnt/nas/music-inbox";
  dataPath = "/var/lib/music-importer";
  port = 8085;
  defaultCategory = "unsorted";
  beetsConfigPath = "/home/user/.config/beets/config.yaml";
  user = "music-importer";
  group = "music-importer";
};
```

### Module creates
- System user/group with permissions on library and inbox paths
- Beets configs: always `inbox-beets.yaml`; `main-beets.yaml` only if `beetsConfigPath` is not set
- systemd service `music-importer.service` running the FastAPI app
- Python environment with dependencies
- Elm build compiled at nix build time, served as static assets

### Data directory
```
{dataPath}/
  main-beets.db       # main library DB (only if app owns config)
  main-beets.yaml     # generated main config (only if app owns config)
  inbox-beets.db      # inbox scratchpad DB (always)
  inbox-beets.yaml    # generated inbox config (always)
  jobs.db             # import job tracking
```

## ytdl-sub Integration (optional, external)

ytdl-sub is one way to feed the inbox. Configure it via the upstream nixpkgs `services.ytdl-sub` module to write audio + `.info.json` sidecars into inbox category subdirectories. The music-importer app has no knowledge of ytdl-sub.

```nix
services.ytdl-sub = {
  user = "music-importer";
  group = "music-importer";

  instances.dj-sets = {
    enable = true;
    schedule = "*-*-* *:00,30:00";
    readWritePaths = [ "/mnt/nas/music-inbox/dj-sets" ];
    config = {
      configuration.working_directory = "/var/lib/ytdl-sub/work";
      presets.audio_inbox = {
        audio_extract = { codec = "mp3"; quality = "320k"; };
        output_options = {
          output_directory = "/mnt/nas/music-inbox/dj-sets";
          file_name = "{upload_date} - {uploader_sanitized} - {title_sanitized}.{ext}";
          maintain_download_archive = true;
        };
        ytdl_options.write_info_json = true;
      };
    };
    subscriptions = {
      "__preset__".preset = "audio_inbox";
      "YT DJ Sets".download.url = "https://www.youtube.com/playlist?list=PLxxx";
      "SC DJ Sets".download.url = "https://soundcloud.com/you/sets/dj-sets";
    };
  };

  instances.live-concerts = {
    enable = true;
    schedule = "*-*-* *:00,30:00";
    readWritePaths = [ "/mnt/nas/music-inbox/live-concerts" ];
    config = {
      configuration.working_directory = "/var/lib/ytdl-sub/work";
      presets.audio_inbox = {
        audio_extract = { codec = "mp3"; quality = "320k"; };
        output_options = {
          output_directory = "/mnt/nas/music-inbox/live-concerts";
          file_name = "{upload_date} - {uploader_sanitized} - {title_sanitized}.{ext}";
          maintain_download_archive = true;
        };
        ytdl_options.write_info_json = true;
      };
    };
    subscriptions = {
      "__preset__".preset = "audio_inbox";
      "YT Concerts".download.url = "https://www.youtube.com/playlist?list=PLyyy";
      "SC Concerts".download.url = "https://soundcloud.com/you/sets/concerts";
    };
  };
};
```

Each ytdl-sub instance maps to a category subdirectory. YouTube and SoundCloud playlists for the same category share an instance — the platform distinction disappears at the inbox boundary.

**Note:** The ytdl-sub preset config is a starting point. Exact field names may vary by version — consult the [ytdl-sub docs](https://ytdl-sub.readthedocs.io/).
